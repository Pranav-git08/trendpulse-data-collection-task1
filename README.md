### TrendPulse - Task 1: Data Collection

### Project Overview
This repository contains the first part of the TrendPulse project. The goal here was to build a script that connects to the HackerNews API and pulls trending stories based on specific categories.

### What this script does:
1. **API Connection**: It uses the `requests` library to fetch the top 500 story IDs from the HackerNews Firebase API.
2. **Filtering**: It looks through those 500 stories and checks the titles for keywords related to:
   - Technology
   - World News
   - Sports
   - Science
   - Entertainment
3. **Data Storage**: Once it finds 25 matches for a category, it moves to the next one. All the results (Title, Author, Score, Category, etc.) are saved into a JSON file.
4. **API Safety**: I added a `time.sleep(2)` delay between categories so the script doesn't hit the API too hard and get blocked.

### Files in this Repo:
* **task1_data_collection.py**: The main Python script used for scraping the data.
* **data/trends_20260405.json**: The raw data file generated by the script.

### How to Run:
1. Make sure you have `requests` installed: `pip install requests`
2. Run the script: `python task1_data_collection.py`
3. The script will create a `data/` folder and save the JSON file there.

### Tech Used:
- Python 3
- Requests Library
- HackerNews API (Firebase)
